# 校园团队任务看板（Trello-lite）PRD

## 1. 背景与目标
面向校园课程/竞赛/实验室小团队，提供一个轻量的任务看板工具，覆盖「工作区 → 项目 → 看板 → 列 → 卡片」的核心链路。

**交付目标（MVP）**：
- 可注册/登录
- 可创建工作区与成员管理（简化邀请：通过邮箱添加）
- 可创建项目与看板
- 看板支持列与卡片的 CRUD
- 卡片支持跨列/同列移动与排序（核心能力）
- 评论、指派、标签（基础版）
- 审计日志（关键写操作可追溯）
- 自动化测试（后端 Jest + E2E Playwright）
- Docker/Compose 一键启动 + GitHub Actions CI

## 2. 角色与权限（RBAC）
工作区级角色：
- **OWNER**：全权限（含转移所有权、删除工作区）
- **ADMIN**：管理成员/项目/看板
- **MEMBER**：日常使用（创建/编辑卡片、评论、移动卡片）

权限规则（MVP）：
- 资源均归属到 workspace；访问前必须验证成员关系
- 只有 OWNER/ADMIN 可管理成员

## 3. 用户故事（含验收标准）
1) 作为用户，我可以注册账号并登录；登录成功后可获取 accessToken。
- AC：注册邮箱唯一；密码加密存储；登录返回 token。
2) 作为用户，我可以创建工作区。
- AC：创建后我成为 OWNER；可在工作区列表中看到。
3) 作为 OWNER/ADMIN，我可以通过邮箱添加成员并指定角色。
- AC：被添加用户必须已注册；重复添加返回 409。
4) 作为成员，我可以在工作区下创建项目。
- AC：项目出现在项目列表；可归档。
5) 作为成员，我可以在项目下创建看板。
- AC：默认生成 3 列（Todo/Doing/Done）。
6) 作为成员，我可以创建列、重命名列、调整列顺序。
- AC：列 position 从 0 开始；同一 board 内 position 唯一。
7) 作为成员，我可以在列中创建卡片并编辑标题/描述/截止时间/优先级。
- AC：卡片在列中按 position 排序。
8) 作为成员，我可以移动卡片到另一列并指定目标位置。
- AC：移动后目标列 position 连续且唯一；原列同样保持连续；接口具备事务性。
9) 作为成员，我可以给卡片添加评论。
- AC：按时间顺序返回；记录 author。
10) 作为成员，我可以给卡片指派负责人、设置标签。
- AC：同一用户不重复指派；标签在 workspace 级复用。
11) 作为 OWNER/ADMIN，我可以查看工作区审计日志。
- AC：能看到谁在何时对哪些实体做了什么（含 meta）。

## 4. 范围边界（不做/延后）
- 不做实时协作（WebSocket）
- 不做复杂通知系统（仅预留）
- 不做全文搜索（可后续扩展）

## 5. 非功能需求
- 安全：JWT、RBAC、登录限流
- 可观测性：结构化日志 + health endpoint
- 可部署：Docker/Compose 一键启动
- 可测试：后端 unit/integration/e2e；前端 E2E 至少 2 条主流程

## 6. 成功指标
- `docker compose up` 可在 10 分钟内完成本地启动并可用
- CI 通过：lint + test + build + e2e（最小集）
